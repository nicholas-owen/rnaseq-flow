Overview

A single pipeline, three axes of choice

rnaseq-flow processes raw FASTQs through QC, alignment, quantification, differential expression, splicing, fusions and enrichment, and aggregates everything into interactive HTML reports.

It is modular in three senses — and you compose them per run.

Aligner--aligner
STAR, HISAT2, Salmon or Kallisto — pick one per run; downstream stages adapt to the choice.
Stages--stop_at, --dtu, --diffsplice
Stop at preQC, postQC, DE or GSEA. Splicing, DTU, fusions and enrichment are opt-in.
References--download_refs, --build_indices
Bring your own FASTA / GTF / indices, or generate them with the bundled helper workflows.

Requirements & install

Setup is four small things

Nextflow itself, a Java runtime, a container engine and a dry-run check. Every pipeline tool lives inside its own container — nothing to install by hand.

01

Install Nextflow

One-liner — drop the binary on your PATH and check the version.

$ curl -s https://get.nextflow.io | bash

Java 11–21

Required by Nextflow. Anything in that range works; check with java -version.

-profile docker | singularity | conda

$ nextflow run main.nf -stub-run ...
04 · Setup

main.nf

Three run modes, one entry point

A single invocation dispatches to one of three workflows. The pipeline runs exactly one mode per execution.

01

Download references

Fetch genome FASTA + GTF (and optional MSigDB GMT gene sets) from Ensembl. Pin a specific release with --download_release; outputs land in a versioned v<release>/ subfolder.

--download_refs

02

Build indices

Build the index for one aligner — or all four at once with --aligner all. Accepts gzipped FASTA / GTF inputs and writes each index into its own directory.

--build_indices

03

Run the analysis

The full RNASEQ workflow: read QC, alignment or pseudo-alignment, quantification, DE, splicing, fusions, enrichment and reporting. Triggered whenever --input is given.

--input samplesheet.csv
05 · main.nf

Stage 1 · Read QC and trimming

From raw FASTQ to clean reads

FastQC profiles raw read quality; fastp trims adapters and low-quality bases; metrics flow on to MultiQC.

Input

FASTQ

Paired- or single-end raw reads, gzipped. Layout auto-detected from the samplesheet.

→
FastQC

Profile

Per-base quality, adapter content, duplication and overrepresented sequences.

→
fastp

Trim + filter

Adapter removal and quality trimming. A large drop in read count signals aggressive filtering.

  • Stop early. --stop_at preQC halts after FastQC; postQC halts after alignment QC — fast end-to-end sanity checks without spending compute on DE.
  • Aggregated, not isolated. All FastQC and fastp metrics are picked up by MultiQC for cross-sample comparison.
06 · Read QC

Stage 2 · Alignment and quantification

Four aligners, one --aligner flag

Two genome aligners (STAR, HISAT2) and two pseudo-aligners (Salmon, Kallisto). Pick one — downstream stages adapt automatically.

STAR
Genome aligner

Default. Sorted BAMs unlock the full analysis: featureCounts, RSeQC, rMATS, STAR-Fusion, diffSplice, coverage tracks.

HISAT2
Genome aligner · lighter

Lower-memory genome alignment. Same downstream stack as STAR minus fusion calling (STAR-Fusion only).

Salmon
Pseudo-aligner

Fast transcript-level quantification with built-in library-type detection. Unlocks isoform switching and DTU.

Kallisto
Pseudo-aligner

Fastest path to transcript abundances. Pair with --strandedness forward/reverse for stranded libraries.

07 · Aligners

Stage 3 · Gene-level counts

Two paths converge on one count matrix

Both routes produce a gene × sample count matrix in the shape DESeq2 and edgeR expect — no manual reshaping.

featureCounts

STAR / HISAT2 route

Counts reads overlapping each gene on the sorted BAM. RSeQC auto-detects strandedness per sample with --strandedness auto, and the verdict feeds straight into -s.

→ featurecounts/<sample>.counts

tximport

Salmon / Kallisto route

Summarises transcript quantifications to gene level with length-aware normalisation. Writes tximport_gene_counts.csv and a txi.rds object DESeq2/edgeR import directly.

→ tximport/txi.rds

08 · Quantification

Stage 4 · Differential expression

DESeq2 and edgeR run in parallel

Two independent callers on the same gene counts — genes called significant by both are the most robust hits.

Input
Gene counts
featureCounts or tximport

DESeq2

Wald test · apeglm LFC shrinkage

  • ~ batch + condition auto-applied when a batch column is present; override with --design.
  • Per-contrast CSVs with gene_name and gene_biotype annotation.
  • MA, volcano and PCA plots; top-20 heatmap.
  • Feeds GSEA (fgsea) and gProfiler downstream.

edgeR

Quasi-likelihood GLM (glmQLFTest)

  • Same design matrix and covariate handling as DESeq2 — comparable methodology.
  • Per-contrast CSVs with logFC, FDR and annotation columns.
  • MDS, smear and per-contrast volcano plots.
  • Independent cross-check — agreement strengthens calls.
Output
Per-contrast
results CSVs+ plots, PCA, heatmap
09 · Differential expression

Stage 5a · Alternative splicing

Two windows on splicing — events and exons

Both run on the genome-aligner route. They answer related but distinct questions, and complement each other.

rMATS

Event-based splicing

Five event classes detected per condition pair:

  • SE — skipped exons
  • MXE — mutually exclusive exons
  • A3SS / A5SS — alternative splice sites
  • RI — retained introns

→ rmats_output/ · cutoff: FDR < 0.05, |ΔPSI| > 0.1

edgeR diffSplice

Differential exon usage

For each exon, tests whether its log-fold-change between conditions departs from the gene's overall log-fold-change.

  • Per-feature table — which exon drives the change
  • Per-gene table — Simes-aggregated across the gene's exons
  • Annotated with gene_name, gene_biotype

opt-in · --diffsplice · needs --gtf

10 · Alt splicing

Stage 5b · Transcript isoforms & fusions

Three views invisible to gene-level DE

A gene can show no change in total expression yet still flip its dominant isoform, redistribute its transcripts, or harbour a fusion.

IsoformSwitchAnalyzeR

Dominant-isoform changes

Flags genes whose dominant transcript flips between conditions — even when total gene expression is flat. Per-gene switch plots included.

Salmon · --isoform_switch

DEXSeq DTU

Transcript-proportion shifts

Tests significant changes in a gene's transcript proportions, independent of total expression. Per-gene q-values aggregate across transcripts.

Salmon / Kallisto · --dtu

STAR-Fusion

Gene-fusion detection

Per-sample fusion_predictions.tsv with junction and spanning-read counts, breakpoints and FFPM. Needs a CTAT library.

STAR · --ctat_lib <dir>

11 · Isoforms & fusions

Stage 6 · Functional enrichment

From gene lists to pathway biology

Two complementary methods downstream of DESeq2 — one ranked, one thresholded. Agreement strengthens the conclusion.

GSEA · fgsea

Ranked-list enrichment

Runs on the full ranked DESeq2 gene list, no cutoffs. Reports NES (direction by sign), p, padj per pathway.

enable with --gmt <file.gmt>

gProfiler · gprofiler2

Over-representation

Tests thresholded significant up- and down-regulated lists separately against GO terms and pathways. Per-contrast tables and gostplot images.

organism via --organism (default hsapiens)

12 · Enrichment

Stage 7 · Reporting

Three reports per run, no clicking required

Every run produces self-contained HTML — start with MultiQC, drill into Quarto, hand off the run summary.

QC aggregation

MultiQC

Single page combining FastQC, fastp, STAR/HISAT2, RSeQC and featureCounts metrics for every sample.

  • Cross-sample comparison and outlier spotting
  • Custom rnaseq-flow theme and ordered modules
  • Best starting point for run-wide QC
multiqc/multiqc_report.html

Analysis narrative

Quarto report

Interactive analysis report that brings every stage's results together in one place; sections appear only when their data is present.

  • Plotly volcano plots (DESeq2 + edgeR)
  • PCA / MDS / heatmap panels
  • Searchable DESeq2, edgeR, GSEA, gProfiler tables
  • DESeq2-vs-edgeR agreement table
quarto_report/analysis_report.html
13 · Reporting

Usage · The input

Everything starts with a four-column CSV

Validated before any process is scheduled — typos and missing files surface immediately.

sample R1 R2 condition
ctrl_1 /data/ctrl_1_R1.fastq.gz /data/ctrl_1_R2.fastq.gz REF
ctrl_2 /data/ctrl_2_R1.fastq.gz /data/ctrl_2_R2.fastq.gz REF
treat_1 /data/treat_1_R1.fastq.gz /data/treat_1_R2.fastq.gz treatment
treat_2 /data/treat_2_R1.fastq.gz /data/treat_2_R2.fastq.gz treatment

REF is the baseline

Naming a group REF makes it the denominator of every contrast — positive log2FC means up in the other condition.

R2 is optional

Single-end vs paired-end is detected automatically from whether the R2 column is populated.

Batch column auto-detected

Add a batch column and the DE model becomes ~ batch + condition; --design takes any formula.

Usage · End-to-end

Scenario A — comprehensive STAR run

Fusions, splicing, differential expression and enrichment, all from a single invocation.

~ / rnaseq-flow
$ nextflow run main.nf \
    --input          samplesheet.csv \
    --aligner        star \
    --star_index     indices/human/star_index \
    --gtf            references/human/annotation.gtf.gz \
    --strandedness   reverse \
    --ctat_lib       /refs/CTAT_lib/ctat_genome_lib_build_dir \
    --diffsplice \
    --gmt            references/human/gmt/c2_kegg.gmt \
    --outdir         results \
    -profile         docker

Salmon variant (Scenario B): swap to --aligner salmon with --salmon_index and --transcript_fasta, add --isoform_switch and --dtu for transcript-level analyses. Same samplesheet, same outputs structure.

Release notes

What's new in 1.1.0

v1.1.0 25 May 2026

Differential splicing — --diffsplice

edgeR diffSpliceDGE at exon level (STAR/HISAT2) or transcript level (Salmon/Kallisto). New diffsplice_output/.

Batch & covariate modelling

A batch samplesheet column is auto-detected; --design accepts any formula such as ~ sex + batch + condition.

Gene-symbol annotation

A new GTF2GENEINFO step adds gene_name and gene_biotype to every DE, DTU and diffSplice result table.

Per-contrast volcano plots

DESeq2 and edgeR each emit a coloured volcano_<A>_vs_<B>.png for every pairwise contrast.

apeglm LFC shrinkage

DESeq2 log2 fold changes are shrunk with lfcShrink; Wald significance is preserved from the unshrunken fit.

Expanded Quarto report + run summary

Full analysis report with interactive plotly volcanoes and searchable tables, plus a self-written run_summary.html after every run.

Quick start — four commands

Validate, fetch, build, run. Each step is a separate invocation of main.nf.

01

Validate the workflow without data

Every process ships a stub block — the full graph dry-runs in seconds.

nextflow run main.nf --input test.csv --aligner star --star_index fake --gtf fake.gtf -stub-run
02

Fetch references for your species

Pin a release; outputs land in a versioned v<release>/ subfolder.

nextflow run main.nf --download_refs --download_species homo_sapiens --download_release 102
03

Build the index for your aligner

Or pass --aligner all to build STAR, HISAT2, Salmon and Kallisto in one run.

nextflow run main.nf --build_indices --aligner star --genome_fasta genome.fa.gz --gtf annotation.gtf.gz
04

Run the analysis

One invocation, end-to-end. Check results/multiqc/multiqc_report.html when it finishes.

nextflow run main.nf --input samplesheet.csv --aligner star --star_index ... --gtf ... -profile docker
